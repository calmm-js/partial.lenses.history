≡ ■ Reference

The combinators provided by this library are available as named imports. Typically one just imports the library as:

import * as H from 'partial.lenses.history'

The examples also use the Partial Lenses library imported as

import * as L from 'partial.lenses'

and the following helper function, thru, that pipes a value through the given sequence of functions:

var thru = (x, ...fns) => fns.reduce((x, fn) => fn(x), x)

≡ ■ Creating

≡ ■ H.init({[maxCount, pushEquals, replacePeriod]}, value) ~> history v0.1.0

H.init creates a new history state object with the given initial value. The named parameters, maxCount, replacePeriod, and pushEquals, are optional and control how history is updated when the state is modified using H.setPresent or via H.viewPresent.

  • maxCount defaults to 2^31-1 and specifies the maximum number of entries to keep in history.
  • pushEquals defaults to false and determines whether writing a value that is equal to the present value updates history or not.
  • replacePeriod defaults to 0 and specifies a period in milliseconds during which an update replaces the present value without adding history.

For example:

H.init({}, 101)

The history data type should be considered opaque. However, assuming that JSON.parse(JSON.stringify(v)) is considered equivalent to v for any value v put into history, then it is guaranteed that JSON.parse(JSON.stringify(history)) is considered equivalent to history.

≡ ■ Present

≡ ■ H.present(history) ~> value v0.1.0

H.present returns the present value from history.

For example:

thru(
  H.init({}, 42),
  H.setPresent(101),
  H.setPresent(69),
  H.undo,
  H.present
)

≡ ■ H.setPresent(value, history) ~> history v0.1.0

H.setPresent sets the present value of history.

For example:

thru(
  H.init({}, 42),
  H.setPresent(101),
  H.present
)

≡ ■ H.viewPresent ~> lens v0.1.0

H.viewPresent is a lens that focuses on the present value of history.

For example:

thru(
  H.init({}, 42),
  L.modify(H.viewPresent, R.negate),
  H.present
)

≡ ■ Undo

≡ ■ H.undo(history) ~> history v0.1.0

H.undo decreases the index of present by one in history if possible.

For example:

thru(
  H.init({}, '1st'),
  H.setPresent('2nd'),
  H.setPresent('3rd'),
  H.undo,
  H.present
)

≡ ■ H.undoCount(history) ~> number v0.1.0

H.undoCount returns the number of times H.undo has an effect starting from present.

For example:

thru(
  H.init({}, '1st'),
  H.setPresent('2nd'),
  H.setPresent('3rd'),
  H.undoCount
)

≡ ■ H.undoForget(history) ~> history v0.1.0

H.undoForget removes all entries prior to present from history.

For example:

thru(
  H.init({}, '1st'),
  H.setPresent('2nd'),
  H.setPresent('3rd'),
  H.undoForget,
  H.undoCount
)

≡ ■ Redo

≡ ■ H.redo(history) ~> history v0.1.0

H.redo increases the index of present by one in history is possible.

For example:

thru(
  H.init({}, '1st'),
  H.setPresent('2nd'),
  H.setPresent('3rd'),
  H.setIndex(0),
  H.redo,
  H.present
)

≡ ■ H.redoForget(history) ~> history v0.1.0
